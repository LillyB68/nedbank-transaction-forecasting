
# Nedbank Transaction Volume Forecasting

## Overview

This repository contains my solution for the **Nedbank Transaction Volume Forecasting Challenge**. The objective of the competition is to develop a machine learning model capable of forecasting customer transaction volumes over a three-month period using historical banking behaviour and transaction data.

This project demonstrates my approach to data preprocessing, exploratory data analysis (EDA), feature engineering, predictive modelling, and model evaluation using Python.

## Project Objectives

* Explore and understand customer transaction behaviour.
* Clean and preprocess banking datasets.
* Engineer meaningful features to improve model performance.
* Build and evaluate forecasting models.
* Generate accurate predictions for competition submission.
